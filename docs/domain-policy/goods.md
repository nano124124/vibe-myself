# Goods 도메인 정책

---

## 0. 개념 정의

| 용어 | 정의 |
|------|------|
| 단일상품 | 옵션이 없는 상품. 재고는 상품 자체에 귀속된다 |
| 옵션상품 | 1개 이상의 옵션 그룹이 연결된 상품. 재고는 옵션 조합(단품) 단위로 관리된다 |
| 단품(Unit) | 옵션 항목 조합 1개를 나타내는 재고/가격 단위. 단일상품은 단품 1개를 가진다 |
| 옵션 그룹 | 상품과 독립적으로 존재하는 분류 기준 (예: 색상, 사이즈). 상품에 연결해서 사용한다 |
| 옵션 항목 | 옵션 그룹에 속하는 개별 값 (예: 빨강, S, M, L) |

---

## 1. 불변조건

> 어떤 오퍼레이션에서도 위반할 수 없는 조건이다.

- 상품은 반드시 **leaf 카테고리**에만 귀속된다 (중간 카테고리 선택 불가)
- **판매가(salePrc)는 0보다 커야** 한다
- **단품 재고는 0 미만이 될 수 없다**
- 이미지는 **최대 5장**이다. 첫 번째 이미지가 대표 이미지다
- 옵션상품의 **단품 옵션 조합은 상품 내에서 유일**해야 한다
- **전시 ON + 판매 가능 상태(SELLING)일 때만** 고객 화면에 노출된다
- 상품은 **물리 삭제하지 않는다**
- **상품 판매 상태는 단품 상태에 종속된다** (단품 재고가 모두 0이면 상품은 SOLD_OUT)

---

## 2. 상태 전이

### 판매 상태

| 상태 | 의미 |
|------|------|
| `SELLING` | 판매 중 |
| `SOLD_OUT` | 품절 (시스템 자동 전이) |
| `STOPPED` | 판매 중지 (관리자 수동 전이) |

**전이 규칙**

```
SELLING  ──재고 모두 0──→  SOLD_OUT
SOLD_OUT ──재고 복구──→    SELLING
SELLING  ──관리자──→       STOPPED
SOLD_OUT ──관리자──→       STOPPED
STOPPED  ──관리자──→       SELLING
```

- `SOLD_OUT`은 관리자가 직접 설정하지 않는다. 재고 변경 시 시스템이 재계산한다
- `STOPPED → SOLD_OUT` 전이는 없다. STOPPED에서 재개하면 재고 기준으로 SELLING 또는 SOLD_OUT이 결정된다

### 전시 상태

| 상태 | 의미 |
|------|------|
| `DISPLAY_ON` | 전시 중 |
| `DISPLAY_OFF` | 전시 안 함 |

- 전시 상태는 관리자가 수동으로 전환한다
- 판매 상태와 독립적으로 존재하지만, 고객 노출은 두 상태를 함께 판단한다

### 품절 판단 기준

- **단품 중 하나라도 재고가 있으면** 상품은 `SELLING` 상태다
- **모든 단품의 재고가 0이면** 상품은 `SOLD_OUT`으로 전이한다
- 재고 판단 단위는 단품(Unit)이다. 상품 단위 재고는 존재하지 않는다

## 3. 오퍼레이션 정책

### 상품 등록

- 카테고리, 판매가, 배송정책은 필수다
- 단일상품으로 등록 시 옵션 없이 단품 1개가 생성된다
- 옵션상품으로 등록 시 선택된 옵션 항목의 직교 조합이 단품으로 자동 생성된다
- 단품의 초기 재고는 0이다. 등록 후 별도로 설정한다
- 전시 태그(`tagNms`)는 선택 항목이다. 태그는 `PR_GOODS_TAG`에 순번(`TAG_SEQ`) 순서로 저장된다
- 이미지 업로드 중 또는 DB insert 중 오류 발생 시 업로드된 이미지는 자동으로 삭제(보상 처리)되고 DB는 롤백된다

### 가격 관리

- 판매가 변경은 `PR_GOODS_PRC` 이력 테이블에 적용 기간(`aplyFromDt` ~ `aplyToDt`)으로 기록된다
- 현재 적용 중인 가격 레코드의 종료일은 `99991231`이다

### 단품 수정

- 재고(`stockQty`), 추가가격(`addPrc`), 사용여부(`useYn`)는 수정 가능하다
- 단품의 옵션 조합 자체는 변경 불가다. 조합 변경이 필요하면 상품 옵션을 수정해야 한다
- 재고 변경 후 상품 판매 상태를 재계산한다

### 상품 삭제

- 물리 삭제는 금지한다
- 판매 상태를 `STOPPED`로 변경하는 것으로 대체한다

## 4. 도메인 경계

> goods 도메인이 책임지지 않는 것을 명시한다.

- **재고 차감은 주문 도메인의 책임이다.** goods 도메인은 재고를 조회하고 상태를 재계산하지만, 차감 트랜잭션은 order 도메인이 주도한다
- **상품 도메인은 최종 결제 가격을 책임지지 않는다.** 프로모션·쿠폰 적용 후 가격은 prom 도메인의 책임이다
- **장바구니·위시리스트 데이터는 상품 도메인이 직접 변경하지 않는다.** 단품 비활성화(`useYn=N`) 또는 상품 `STOPPED` 전환 시에도 cart·wishlist 레코드는 그대로 유지된다. 조회 시 상품·단품 상태 기준으로 필터링하는 것은 cart 도메인의 책임이다
