# 모듈 정의

서비스를 구성하는 도메인 모듈의 명칭과 책임 범위를 정의한다.

## 모듈 목록

| 한국어 | 모듈명 | 책임 범위 |
|--------|--------|-----------|
| 회원 | `member` | 회원가입, 로그인, 프로필, 인증/인가 |
| 상품 | `goods` | 상품 등록/수정/삭제, 상품 상세, 재고 |
| 장바구니 | `cart` | 장바구니 담기/수정/삭제 |
| 주문/클레임 | `order` | 주문 생성/조회, 취소/교환/반품(클레임) |
| 프로모션/이벤트 | `prom` | 쿠폰, 할인, 이벤트 |

## 핵심 규칙

> API 또는 화면을 생성할 때는 반드시 해당 모듈의 패키지/폴더 하위에 파일을 생성한다.
> 모듈 경계를 넘는 로직은 별도 협의 없이 임의로 섞지 않는다.

- Backend 구조 → `@docs/backend/structure.md`
- Frontend 구조 → `@docs/frontend/structure.md`
