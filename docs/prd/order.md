# 주문/클레임 (Order & Claim)

> 주문 생성/조회, 결제(PG 연동), 배송비, 취소/반품/교환(클레임) 처리 기능을 포함한다.
> 클레임은 주문상품 단위로 처리하며, 주문상품의 처리 이력(주문/클레임/철회)을 row로 관리한다.
> 배송지 관리는 회원(ET) 모듈에서 처리한다.

---

## 주문 관리

### 주문 목록 조회 (어드민)
- [ ] 주문 목록을 조회할 수 있다
- [ ] 주문번호, 주문자명, 주문상태, 주문일자로 검색할 수 있다

### 주문 상세 조회 (어드민)
- [ ] 주문 기본정보(주문자, 수령자, 배송지)를 조회할 수 있다
- [ ] 주문상품별 처리 이력(주문 → 클레임 → 철회)을 조회할 수 있다
- [ ] 배송비 내역을 조회할 수 있다
- [ ] 프로모션/쿠폰 적용 내역을 조회할 수 있다
- [ ] 결제 정보 및 결제수단별 배분 내역을 조회할 수 있다
- [ ] 송장번호를 등록할 수 있다

### 주문 생성 (고객)
- [ ] 장바구니에서 선택한 상품으로 주문을 생성할 수 있다
- [ ] 저장된 배송지를 선택하거나 새 배송지를 입력할 수 있다
- [ ] 배송 메모를 입력할 수 있다
- [ ] 적용 가능한 쿠폰/프로모션을 선택할 수 있다
- [ ] 결제 수단을 선택하여 결제할 수 있다
- [ ] 주문 완료 후 주문 상세 화면으로 이동한다

### 주문 목록 조회 (고객)
- [ ] 본인의 주문 목록을 조회할 수 있다
- [ ] 주문상태별로 필터링할 수 있다

### 주문 상세 조회 (고객)
- [ ] 주문 기본정보, 상품, 배송비, 결제 정보를 조회할 수 있다
- [ ] 주문상품별 현재 처리 상태를 조회할 수 있다

---

## 결제 관리

### 결제 처리
- [ ] PG사를 통해 결제를 요청할 수 있다
- [ ] 결제 승인 결과(PG 거래번호, 승인번호 등)를 저장한다
- [ ] 결제 수단은 신용카드, 무통장입금을 지원한다
- [ ] 무통장입금의 경우 입금 확인 처리를 할 수 있다 (어드민)

### 결제 취소 처리
- [ ] 클레임 승인 시 PG사를 통해 결제 취소(환불)를 요청한다
- [ ] 주문상품 단위 부분 취소를 지원한다
- [ ] 결제 취소 결과를 저장한다

### 결제 내역 조회 (어드민)
- [ ] 결제 내역을 조회할 수 있다
- [ ] 결제수단, 결제상태, 결제일자로 검색할 수 있다

---

## 클레임 관리

> 클레임은 주문상품 단위로 신청한다.
> 교환/반품은 철회할 수 있으며, 철회도 이력 row로 관리한다.

### 취소
- [ ] 배송 시작 전 주문상품 단위로 취소를 신청할 수 있다 (고객)
- [ ] 취소 요청을 승인/거부할 수 있다 (어드민)
- [ ] 취소 승인 시 해당 상품 금액을 PG 부분 취소로 환불 처리한다

### 반품
- [ ] 배송 완료 후 주문상품 단위로 반품을 신청할 수 있다 (고객)
- [ ] 반품 사유를 입력할 수 있다
- [ ] 반품 요청을 승인/거부할 수 있다 (어드민)
- [ ] 반품 승인 시 해당 상품 금액을 PG 부분 취소로 환불 처리한다
- [ ] 반품을 철회할 수 있다 (고객, 반품 신청 후 승인 전)

### 교환
- [ ] 배송 완료 후 주문상품 단위로 교환을 신청할 수 있다 (고객)
- [ ] 교환 사유와 교환할 단품을 선택할 수 있다
- [ ] 교환 요청을 승인/거부할 수 있다 (어드민)
- [ ] 교환을 철회할 수 있다 (고객, 교환 신청 후 승인 전)
